---
title: "Close Session"
api: "POST /api/v1/session/closesession"
description: "Close the session associated with the current API key"
---

Close the session associated with the current API key.

This endpoint terminates the active session and settles any remaining payments with the provider. Any unused session time may be refunded according to the session terms.

### Headers

<ParamField header="Authorization" type="string" required>
  API key in format: `Bearer sk-xxxxxx`
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Whether the session was successfully closed
</ResponseField>

<ResponseField name="session_id" type="string">
  ID of the closed session
</ResponseField>

<ResponseField name="refund" type="string">
  Amount refunded for unused session time (if applicable)
</ResponseField>

<ResponseField name="total_usage" type="object">
  Usage statistics for the session
</ResponseField>

<RequestExample>
```bash cURL
curl --request POST \
  --url https://api.dev.mor.org/api/v1/session/closesession \
  --header 'Authorization: Bearer sk-YOUR_API_KEY'
```

<Info>
  Sessions automatically close when they expire. You only need to manually close a session if you want to terminate it early or switch to a different provider.
</Info>

<Note>
  After closing a session, you can immediately create a new session with the same or different provider/model.
</Note>
