---
title: "Get Private Key Status"
api: "GET /api/v1/auth/private-key"
description: "Check if a user has a private key registered"
---

Check if a user has a private key registered.

Does not return the actual key, only status information. This is useful for checking whether a user needs to register their private key before performing blockchain operations.

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token (JWT) from OAuth2 login
</ParamField>

### Response

<ResponseField name="has_key" type="boolean">
  Whether the user has a private key registered
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp when the key was first stored (null if no key)
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp when the key was last updated (null if no key)
</ResponseField>

<Info>
  **Security**: This endpoint only returns status information, never the actual private key. The private key is stored encrypted and can only be used for blockchain operations within the gateway.
</Info>

<Note>
  A private key is required for blockchain operations like approving token spending and creating sessions with providers.
</Note>
